# Release v0.1.8

## Added

- OpenClaw runtime interoperability foundation:
  - plugin adapter + skill pack
  - `POST /tools/invoke` bridge tool
  - OpenClaw agent-routing header support
- Runtime HTTP control-plane endpoints for execute, run summary, pointer summary, and run resume.
- Crash-safe checkpoint resume/replay, plus MCP tool bridge interoperability.
- Evaluation hardening:
  - CI eval scorecard gate
  - CI competitive benchmark suite
  - CI soak + chaos reliability gate with threshold and baseline regression checks
- Operator and adoption tooling:
  - one-command bootstrap + smoke artifact
  - guided doctor diagnostics
  - self-hosted control-plane reference stack
  - telemetry export pack
  - state backup/restore workflow and runbook
- Reliability/security governance controls:
  - deny-by-default runtime policy profile templates
  - remote-ops auth scopes and ownership governance
  - runtime rate limits and concurrency guardrails
  - branch-expression evaluation hardening
  - strict typed-surface gate for selected high-risk runtime modules

## Changed

- README was rewritten to align positioning with shipped reliability and interop capabilities.
- Proof sheet and architecture/tooling docs were expanded and contract-checked in CI.
- V1 compatibility contract checks are now enforced in CI and reflected in the release workflow.

## Fixed

- CI regressions in scoped strict typing gate were corrected.
- Multiple doc and operational runbooks were tightened to reflect current runtime behavior.

## Deprecated

- None.

## Removed

- None.

## Breaking Changes

- None intended in this release.

## Upgrade Notes

- Upgrade package version to `0.1.8`.
- Re-run local bootstrap and diagnostics:
  - `./scripts/bootstrap_local.sh`
  - `python scripts/eap_doctor.py doctor --env-file .env --output-json artifacts/doctor/diagnostics.json`
- If using OpenClaw interop, validate gateway and plugin paths:
  - `./scripts/interop_openclaw_smoke.sh v2026.2.22`
- If operating in production-like environments, validate gates and ops controls:
  - `python scripts/eval_scorecard.py --output-dir artifacts/eval --threshold-config docs/eval_thresholds.json --baseline docs/eval_baseline.json`
  - `python scripts/soak_chaos_scorecard.py --output-dir artifacts/soak_chaos --threshold-config docs/soak_chaos_thresholds.json --baseline docs/soak_chaos_baseline.json`
